# Respuestas Teóricas

## Pregunta 1
El archivo facilita la comprensión a la persona que analice este código por si lo quiere implementar para el uso que requiera, o hasta modificarlo para poder mejorar o hacerlo parte de algo más grande. Además, actúa como un manual técnico fundamental para quien no estuvo desde el inicio, detallando las características del entorno (sistema operativo, versión de C++, compilador) y las instrucciones exactas de compilación y funcionamiento del programa (GitHub Docs, s.f.).

## Pregunta 2
Un Makefile sirve para organizar todo lo necesario para compilar y enlazar este software, ayudando a que el que lo revise lo pruebe más fácilmente (GeeksforGeeks, s.f.). Aunque Make haya sido creado para C, se puede usar en cualquier lenguaje. Sí, es necesario tener consideraciones especiales, como el hecho de que se comporta nativamente en Linux y macOS, y que los comandos deben empezar con tabulador, entre otras cosas.

## Pregunta 3
Usar tecnologías como Git, en especial GitHub, nos da un historial de los cambios y una edición fácil para cualquier persona, algo que facilita significativamente los proyectos (Hostinger, s.f.). En trabajos de 3 o más personas, todas estas herramientas agilizan el análisis y edición del proyecto.

## Pregunta 4
El código es algo bastante personal donde cada uno tiene una manera de pensar, por lo que leer el código de alguien puede ser un trabajo arduo. Documentar funciones nos ayuda a tener un análisis mucho más rápido; por lo que no documentar funciones cierra posibilidades de obtener feedback de otras personas o complejiza el trabajo en equipo (Beecrowd, s.f.).

## Pregunta 5
Estas herramientas se complementan para un desarrollo amigable con usuarios ajenos, ya que representan diferentes capas de profundidad en el funcionamiento del código. GitHub proporciona el acceso y control de versiones; la documentación de funciones detalla la lógica interna; el Makefile automatiza la compilación y el enlace del código; y el README.md conecta todos estos elementos al documentar cómo usar las demás herramientas (Drake, 2020).
